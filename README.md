# InferenceDeck ---面向多组学癌症分型的全栈AI分析平台（副标题暂定）




这个平台是干什么的？
我们这个平台主要受众是研究者，他们的论文里经常要比较各种算法，所以就可以来我们这个平台，使用自己的输入数据，选择并比较各种经典算法，得到详细的结果，然后再手动测一次自己的算法就行了。
也就是说我们这个平台作用是减少研究者的工作量，省去了搭建对照组的时间。

哈哈在这之前我一直以为我们这个平台服务的是普通一线医生...
不过这样也好，想象了一下，如果主要受众是研究者，并且作用还是实打实地减少他们写论文时的工作量，再加上老师的宣传，那么使用的人数首先就有一个基础了。毕竟这样真的能实打实地方便研究者。





开发一个“可选择算法的多组学/多模型分析平台”，支持用户上传数据并选择不同机器学习/深度学习/多组学聚类算法进行分析，提供可视化结果与报告导出功能。

一个面向非技术用户的低代码AI分析SaaS平台，支持用户上传数据、动态选择多模态算法（聚类/分类/回归）并进行可视化分析。



该项目由两个人开发，我目前的打算是我负责整个平台，以及其中一个算法的实现；另一位协作者实现剩下的算法


因为注释详细所以适合学习？


打算锻炼一下我的前后端分离开发能力，所以技术栈我选择的是vue+FasetAPI。

因为使用的是前后端分离模式，所以运行这个项目需要同时运行两个cmd窗口


backend文件夹里是后端代码
frontend文件夹里是前端代码


如何运行？
运行后端：cd到backend文件夹，conda activate myenv，python server.py
运行前端：cd到frontend文件夹，npm run dev
